# lib/

Three utility modules shared across all dashboard pages.

---

## accounts.ts

Static list of the five Hardhat demo identities used in the account picker.

```typescript
type Account = {
  name: string;       // display name (e.g. "TAP Air Cargo")
  address: string;    // Ethereum address (0x…)
  privateKey: string; // Hardhat dev key — safe only for local demo
};
```

| Index | Name | Role |
|-------|------|------|
| 0 | IATA | Admin / oracle owner |
| 1 | TAP Air Cargo | Carrier A |
| 2 | Pfizer | Shipper |
| 3 | DHL Aviation | Carrier B |
| 4 | MSF Luanda | Receiver |

These are standard Hardhat deterministic accounts. The private keys are
**publicly known** and must never be used on any real network.

## signer.ts

Returns an `ethers.Signer` for the currently selected demo identity (or
MetaMask if available).

```typescript
getSigner(): Promise<ethers.Signer>
```

**Logic:**
1. If `window.ethereum` is present (MetaMask), return a `BrowserProvider` signer
2. Otherwise, read the selected account from `localStorage` (set by the
   `AccountPicker` component) and return a `Wallet` connected to the Hardhat RPC

Used by every dashboard page to sign contract transactions.

## errors.ts

Maps Solidity custom-error 4-byte selectors to human-readable strings so the
UI can display meaningful error messages instead of raw hex reverts.

```typescript
decodeError(error: unknown): string
```

| Custom error | Selector | Message shown |
|-------------|---------|--------------|
| `NotCurrentHolder()` | `0x…` | "You are not the current package holder" |
| `AlreadyDelivered()` | `0x…` | "This package has already been delivered" |
| `InvalidRecipient()` | `0x…` | "Recipient address cannot be zero" |
| `NotOracle()` | `0x…` | "Caller is not an approved oracle" |
| `NotAFactoryPackage()` | `0x…` | "Package ID does not exist in this factory" |

The selectors are generated by `keccak256("ErrorName()")` and must match the
contracts exactly. The `test/Errors.test.ts` file verifies this alignment.
