# app/

Next.js 14 frontend for CargoChain. Four role-based dashboards, no wallet
extension required — a built-in account picker lets you sign as any Hardhat
demo identity.

```bash
cd prototype/app
npm run dev        # http://localhost:3000
npm run build      # production build
npm run start      # serve production build
```

---

## Tech Stack

| Library | Version | Purpose |
|---------|---------|---------|
| Next.js | 14 | React framework + file-based routing |
| ethers.js | 6 | EVM interaction (contract calls, event subscriptions) |
| Tailwind CSS | 3 | Utility-first styling |
| TypeScript | 5 | Type safety across all pages and libs |

---

## Pages

| Route | Dashboard | Actor | Key Actions |
|-------|-----------|-------|-------------|
| `/` | Home | — | Role picker landing page |
| `/shipper` | Shipper | Pfizer | Create package via `PackageFactory.create()` |
| `/carrier` | Carrier | TAP / DHL / MSF | `Package.transferCustody()` or `markDelivered()` |
| `/simulation` | Simulation | Anyone | Watch IoT batches appear live; verify any reading on-chain |
| `/regulator` | Regulator | IATA / Anyone | Full audit trail for any package ID |

See [pages/README.md](pages/README.md) for per-dashboard details.

## Environment Variables

Set in `prototype/app/.env.local` (auto-written by `scripts/deploy.ts`):

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_RPC` | Hardhat node RPC (`http://127.0.0.1:8545`) |
| `NEXT_PUBLIC_CHAIN_ID` | Chain ID (`31337` for local Hardhat) |
| `NEXT_PUBLIC_FACTORY` | Deployed `PackageFactory` address |
| `NEXT_PUBLIC_MERKLE` | Deployed `MerkleIoT` address |

If `.env.local` is missing, re-run `npm run deploy:local` from `prototype/` —
it writes the file automatically.

## Account Picker (no wallet needed)

The `AccountPicker` component in the top-right of every dashboard lets you
switch between demo identities without a wallet extension:

| Identity | Role | Hardhat Account |
|----------|------|----------------|
| IATA | Admin / oracle owner | #0 |
| TAP Air Cargo | Carrier A | #1 |
| Pfizer | Shipper | #2 |
| DHL Aviation | Carrier B | #3 |
| MSF Luanda | Receiver | #4 |

Selection persists in `localStorage` — survives page refreshes and navigation.
If MetaMask is detected and connected to the Hardhat network, the app uses
MetaMask instead of the picker automatically.

## IoT Batch Files

The `oracle-simulator.ts` script writes JSON files to:
```
prototype/app/public/oracle-batches/batch-<id>.json
```

Next.js serves these as static files at `/oracle-batches/batch-<id>.json`.
The Simulation dashboard fetches them to display readings and pass Merkle proofs
to `MerkleIoT.verifyReading()`.

## Adding a New Dashboard Page

1. Create `pages/your-page.tsx` — export a default React component
2. Add an entry to the dashboards array in `pages/index.tsx`
3. Import `lib/signer.ts` to get an ethers signer for contract calls
4. Import the ABI from `typechain-types/` (generated by `npx hardhat compile`)
